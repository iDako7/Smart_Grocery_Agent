## Deviations from Spec

### D1 — Linked meal plan not created on grocery list save *(open)*

**Observed:** `POST /saved/grocery-lists` inserts only into `saved_grocery_lists`. No row inserted into `saved_meal_plans`. The two tables have no FK relationship (`src/backend/db/tables.py` — `saved_grocery_lists` has no `meal_plan_id` column).

**Expected (spec §3 Journey 1):** "Save: grocery list and associated meal plan persisted as linked pair, both viewable from sidebar."

**Implementation gap:** `src/backend/api/saved.py:285-305` — `create_grocery_list` reads `session.state_snapshot["grocery_list"]` but does not create a meal plan record.

**Status:** Open — separate issue to be filed.

---

### D2 — Agent does not transition to recipe cards after clarify confirmation *(FIXED — PR #99/#101)*

**Was observed (2026-04-16):** On turn 2 (screen=`clarify`), agent called `search_recipes` internally then chose `emit_clarify_turn(questions=[])` instead of emitting `recipe_card` events.

**Root cause:** `src/ai/orchestrator.py` saved `response_text=""` to conversation history for clarify turns. On the next `/chat` call the LLM had no context of the PCSV analysis or questions asked, causing stochastic failure to proceed to recipe cards.

**Fix (commit `5df2a6d`):**
- Added `ClarifyTurnPayload.to_context_text()` in `contracts/tool_schemas.py` — serializes clarify payload into a compact LLM-readable string
- Updated 3 return paths in `src/ai/orchestrator.py` to use this instead of `""`
- Added regression tests: `tests/test_clarify_context_text.py` (guards `to_context_text()` never returns `""`)

**Verified fixed:** 2026-04-17 run — 4 recipe_card events received on turn 2, all with valid id + name.
