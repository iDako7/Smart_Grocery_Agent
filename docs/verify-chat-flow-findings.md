## Deviations from Spec

### D1 — Linked meal plan not created on grocery list save

**Observed:** `POST /saved/grocery-lists` inserts a row into `saved_grocery_lists` and returns a `SavedGroceryList`. No row is inserted into `saved_meal_plans`. The two tables have no FK relationship (confirmed in `src/backend/db/tables.py` — `saved_grocery_lists` has no `meal_plan_id` column).

**Expected (spec §3 Journey 1):** "Save: grocery list and associated meal plan persisted as linked pair, both viewable from sidebar."

**Expected (spec §2 G3):** "The saved grocery list references its associated meal plan so the user can navigate between them."

**Implementation gap:** `src/backend/api/saved.py:284-305` — the `create_grocery_list` handler reads `session.state_snapshot["grocery_list"]` for stores data but does not create a meal plan record or store any `meal_plan_id` reference.

**Impact:** After saving, the sidebar Meal Plans section does not gain an entry; there is no navigation from saved grocery list → meal plan.

**Proposed follow-up:** File a new issue for implementing linked-pair persistence. Not in scope for #107.

### D2 — Agent does not transition to recipe cards after clarify confirmation

**Observed:** On turn 2 (screen=`clarify`), with user message `"Looks good, show me recipes. Noodles. Garlic soy."`:
- Agent internally called `search_recipes` (confirmed by `thinking` event)
- Agent then called `emit_clarify_turn` with `questions=[]`
- Result: `clarify_turn` SSE event emitted (no `recipe_card` events)
- `done.status = complete` — this is not an error, the agent made a deliberate choice

**Expected (spec §3 Journey 1):** After user confirms on Clarify screen, the response should include 3–5 recipe cards covering PCV gaps.

**Root cause hypothesis:** The LLM's decision to call `emit_clarify_turn` vs proceed to recipe display is controlled by the system prompt's clarify screen instructions. The current prompt may not reliably trigger the transition when the user says "show me recipes" but a PCSV `sauce` gap still exists. Distinct from issue #87 (crash/error path) — this is a behavioral path issue.

**Impact:** Journey 1 cannot complete through the happy path. The clarify → recipes transition is a core part of the flow.

**Proposed follow-up:** Investigate the clarify screen system prompt rules. Likely needs a prompt update to specify that when the user explicitly requests recipes, `search_recipes` results should be emitted as `recipe_card` events regardless of remaining gaps. Could be a fix within #87's scope or a separate prompt engineering issue.
