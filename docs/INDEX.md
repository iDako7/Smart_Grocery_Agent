# Docs Index

This folder contains the core planning and architecture documents for Smart Grocery Assistant V2.

## Documents

| File | Purpose |
|---|---|
| `product-spec-v2.md` | Product vision, target users, core principles, user stories, and feature scope. |
| `architecture-spec-v2.md` | End-to-end system architecture, major components, data flow, interfaces, and open questions. |
| `ai-layer-architecture-v2.md` | Internal AI-layer design, orchestration model, tool architecture, and ADRs. |
| `Implementation_Plan_high_level.md` | High-level phased roadmap across Phase 1, Phase 2, and Phase 3. |
| `phase-1-plan.md` | Detailed execution plan for Phase 1, including harness build, prompt iteration, and Phase 2 handoff. |

## Suggested Reading Order

1. `product-spec-v2.md`
2. `Implementation_Plan_high_level.md`
3. `phase-1-plan.md`
4. `architecture-spec-v2.md`
5. `ai-layer-architecture-v2.md`

## Notes

- `product-spec-v2.md` explains what the product should do.
- `Implementation_Plan_high_level.md` explains the overall delivery sequence.
- `phase-1-plan.md` expands the first phase into concrete execution steps.
- `architecture-spec-v2.md` explains how the system is intended to fit together.
- `ai-layer-architecture-v2.md` goes deeper on the agent runtime, tools, and architecture decisions.
